openclawApr 6
RELEASE

openclaw breaks its own config, on purpose, with a migration path

Legacy config aliases like 'talk.voiceId' and 'browser.ssrfPolicy.allowPrivateNetwork' are gone — but 'openclaw doctor --fix' will rewrite your config for you.

Config sprawl accumulates quietly: aliases that made sense three versions ago stick around because nobody wants to break existing deployments. v2026.4.5 finally rips out a batch of them — 'talk.voiceId', 'talk.apiKey', 'agents.*.sandbox.perSession', 'hooks.internal.handlers', and the channel/group/room 'allow' toggles all resolve to their canonical paths now. The breaking change ships with load-time compatibility shims and an 'openclaw doctor --fix' migration command (#60726, thanks @vincentkoc), so existing configs won't hard-fail on upgrade. The same release also lands built-in 'video_generate' and 'music_generate' tools with bundled Google Lyria and MiniMax providers, plus a ComfyUI workflow plugin for local image/video/music generation — turning openclaw into something closer to a media production router than a chatbot.

openclawApr 8
RELEASE

every bundled channel was importing from paths that don't exist after npm install

Telegram, Slack, Matrix, Teams, and eight other channels all tried to load from 'dist/extensions/*/src/*' — a path only present in development builds.

If you installed openclaw via npm rather than running from source, gateway startup could fail silently because bundled channel plugins were importing setup and secret contracts from development-only paths like 'dist/extensions/telegram/src/*'. v2026.4.7 and v2026.4.8 fix this across the board — Telegram first, then BlueBubbles, Feishu, Google Chat, IRC, Matrix, Mattermost, Microsoft Teams, Nextcloud Talk, Slack, and Zalo — by routing those imports through packaged top-level sidecars. Separately, v2026.4.7 introduces 'openclaw infer', a first-class CLI hub for provider-backed inference across model, media, web, and embedding tasks, and adds pluggable compaction providers (#56224) so you can replace the built-in summarization pipeline entirely.

openclawApr 8
RELEASE

Slack's socket mode now respects your proxy — no monkey patch required

Proxy-only deployments had to patch around openclaw's WebSocket connections; ambient HTTP(S)_PROXY and NO_PROXY settings are finally honored.

Deploying openclaw behind a corporate proxy meant Slack's Socket Mode WebSocket connections ignored ambient proxy settings entirely, forcing operators to monkey-patch the connection layer. v2026.4.8 (#62878, thanks @mjamiv) threads HTTP(S)_PROXY and NO_PROXY through Socket Mode properly. The same release fixes a subtler Slack issue: when bot tokens used SecretRef-backed config, 'downloadFile' was re-reading raw config instead of using the already-resolved token, causing file downloads to fail (#62097, thanks @martingarramon). Both fixes land alongside a network-layer change that skips DNS pinning when a trusted env-proxy is active (#59007), letting sandbox environments delegate host resolution to the proxy.

summarizeApr 7
RELEASE

summarize v0.13.0 extracts slides from local video files, not just URLs

The '--slides' flag now works on files sitting on your disk — no upload required, no fake 'Downloading audio' progress bar.

The slide extraction pipeline in summarize only worked with remote URLs, so local conference recordings or screen captures had to be uploaded somewhere first. v0.13.0 (#149, by @steipete) routes local video files through the same slide-aware flow, fixing the awkward 'Downloading audio' progress text that appeared even when nothing was being downloaded. The release also adds GitHub Copilot as a first-class model provider — 'github-copilot/gpt-5.4' and friends authenticate via GITHUB_TOKEN — and adds OpenCode as a CLI provider alongside existing OpenClaw support. GPT-5.4 family routing got a quiet fix too: requests were hitting a stale generic parser instead of direct provider APIs, and GitHub Models shorthand inference was pinning old prefixes instead of normalizing newer model IDs like 'o5' and 'claude-opus-4.6'.

openclawApr 8
FEATURE

session compaction gets checkpoints — and a UI to rewind them

Operators can now inspect and restore pre-compaction session state instead of trusting the summarizer got it right.

Long-running agent sessions get compacted to keep context windows manageable, but once the summarization ran, the original state was gone. v2026.4.7 (#62146, thanks @scoootscooob) adds persisted compaction checkpoints with branch/restore actions in the Sessions UI. If a compaction mangled important context, operators can roll back. This pairs with the new pluggable compaction provider registry (#56224) — if the built-in LLM summarization isn't good enough, you can swap in your own pipeline and still have checkpoint safety underneath.

openclawApr 8
RELEASE

the exec tool was lying about its sandbox policy

'host=auto' sessions reported stale stricter defaults instead of the actual host-aware fallback — 'full/off' on gateway, 'deny/off' on sandbox.

The '/exec' endpoint's current-default reporting was out of sync with real runtime behavior. When 'host=auto' was configured, the tool surfaced an older, stricter default policy instead of the actual fallback: 'full/off' on gateway or node hosts, 'deny/off' on sandbox hosts. v2026.4.8 corrects the reporting so operators debugging exec permissions see what's actually enforced, not what used to be enforced. A small fix, but the kind of silent lie that burns an hour of debugging when a sandbox exec call fails and the reported policy says it shouldn't.
