FEATURE

subscribed threads now respect opt-out decisions

*Junior's routing layer stops re-engaging users who've already declined.*

@dcramer's #107 adds explicit opt-out tracking to the subscribed-thread routing path. Previously, the system had no way to honor a user's decision to exit a conversation thread; it would re-route and re-engage regardless. Now opt-out decisions flow through the decision layer, and threads stay dismissed when users close them. Combined with #95 (which preserves DM and mention routing at ingress), junior's message dispatch is now more predictable.

RELEASE

cloudflare workers now emit response and rate-limit metrics

*Visibility into MCP server performance under load.*

@dcramer's #848 instruments the cloudflare workers runtime with response time and rate-limit metrics. The MCP server was previously running blind—no way to observe latency or throttling behavior in production. These metrics now flow into observability systems, making it clear when the server is under pressure or misbehaving. Version 0.30.0 ships with this in place. Separately, #839 removes root protected resource metadata that was leaking into the API surface.

FEATURE

test harness now runs cloudflare workers tests in workers pool

*Tests execute in the actual runtime environment instead of Node.js.*

@dcramer's #678 moves cloudflare workers tests from a generic Node.js vitest runner to vitest-pool-workers, which executes them inside the actual cloudflare workers runtime. This catches environment-specific bugs (bindings, globals, async behavior) that a Node.js simulation would miss. The test suite now reflects production reality.

FEATURE

bottle precision now tracks whisky release dates

*Distinguishing bottlings by age and vintage expression becomes automatic.*

@dcramer's #295 adds release-aware bottle precision to the whisky schema. Previously, two bottles from the same distillery at the same ABV couldn't be reliably differentiated without manual curation. Now the system tracks release date metadata and uses it to generate correct precision identifiers. The four open issues (schema revamp, distillery name mapping, batch queries, org support) represent the longer roadmap but are separate from this week's work.

auth guidance now surfaces through the skill layer generically

*No more hardcoding credential messages per tool.*

@dcramer's #106 lifts auth guidance out of individual skill implementations and surfaces it through a generic mechanism. When a tool needs credentials, users see consistent, standardized help text instead of whatever prose the skill author happened to write. This pairs with #93, which adds plugin-scoped MCP tool support—skills can now declare tools that are only available within their own scope, reducing namespace collisions and permission complexity.

FEATURE

queue diagnostics now track silent thread pool drops

*When messages disappear without error, you'll know why.*

@dcramer's #92 adds diagnostic logging to the message queue and thread pool shutdown path. Junior was silently dropping queued messages when threads exited, with no trace in logs. Now the system records what was discarded and where in the shutdown sequence it happened. #96 hardens the eval harness overrides and updates dependencies to prevent similar silent failures downstream.

FEATURE

image attachments now reuse generated images across tool invocations

*File attachment dedupe cuts redundant I/O and storage.*

@dcramer's #91 caches generated images and reuses them when the same visual is needed across multiple tool calls in a single message thread. Previously, each tool invocation would regenerate and re-upload the same image. Now the cache key is checked first, saving bandwidth and storage quota during longer interactions.
