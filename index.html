RELEASE

rpg v0.8.0 Ships EXPLAIN Visualization That Spots Your Bad Plans Before You Do

The Rust-based Postgres terminal now colorizes query plans, highlights hot paths, and flags seq scans on large tables automatically.

Four releases in five days. That is not a typo. rpg went from v0.5.0 to v0.8.0 between March 17 and March 20, shipping a new version roughly every 18 hours. The headline feature in v0.8.0 is an enhanced EXPLAIN visualizer that detects row estimate errors exceeding 10x, sorts spilling to disk, hash batches greater than one, and parallel workers that failed to launch. A summary header displays execution time, planning time, rows, and buffer stats at a glance. Users can toggle between psql-compatible raw output and the new enhanced view with \pset explain_format. The release also includes EXPLAIN plan sharing to depesz and dalibo, plus a new \s command for history display and save.

FEATURE

rpg Locks Down Its AI: /ask Queries Now Run Read-Only, Period

Versions 0.6.0 and 0.7.0 wrapped every AI-generated query in a read-only transaction and added auto-rollback to prevent idle-in-tx pileups.

The back-to-back v0.6.0 and v0.7.0 releases focused squarely on safety. Every query executed by the /ask command is now wrapped in a read-only transaction at the database level, blocking DDL and DML even if query classification fails. Stale transactions from errors or cancellations are automatically rolled back. Ctrl-C now cancels in-flight LLM API calls. The text2sql mode (\t2s) retains write capability but requires explicit user confirmation with an Execute write query? [y/N/e] prompt.

SECURITY

AI Agent Leaks pgMustard API Key in Public PR Comment

The key was rotated quickly, but the lesson is clear: let your agent post to GitHub unsupervised at your own peril.

During development of the new \explain share pgmustard feature in v0.8.0, an AI agent helpfully dumped its testing output into a public PR comment — API key and all. The credential was spotted fast and rotated before any known misuse. No data was compromised. The incident is minor in impact but rich in instruction: agents that post to public repositories need credential scanning in their output pipeline. If your robot can push to GitHub, your robot can leak to GitHub.

RELEASE

v0.5.0 Brings Syntax Highlighting and Auto-Detects Your AI Provider

Keywords get colors, errors turn red, and rpg now sniffs ANTHROPIC_API_KEY, OPENAI_API_KEY, or OLLAMA_API_KEY from your environment without configuration.

The March 17 release added multi-category SQL syntax highlighting with distinct colors for keywords, strings, and operators. Error prefixes are now colorized: red for ERROR, yellow for WARNING, cyan for NOTICE. The AI provider is auto-detected from environment variables, removing a setup step. Autonomous agent references were scrubbed from the codebase and README — rpg is positioning itself as a focused terminal tool with light AI integration, not an agent framework.
